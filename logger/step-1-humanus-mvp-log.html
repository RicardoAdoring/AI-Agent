四、当前核心模块说明

文件 职责
app/main.py 创建 FastAPI 应用，配置 CORS，注册 AI 路由，提供 /health 健康检查。
app/api/ai.py 提供与原 Vue 前端兼容的聊天接口，包括同步聊天、SSE 聊天和 Manus 占位流式接口。
app/core/config.py 通过 pydantic-settings 读取 .env 配置，包括主模型、备用模型、聊天记忆目录等。
app/llm/factory.py 根据配置创建主模型或 Ollama 备用模型。主模型使用 langchain_openai.ChatOpenAI 连接本地 OpenAI-compatible 服务。
app/memory/file_chat_memory.py 实现基于 JSON 文件的多轮对话记忆，并对 chatId 做安全清洗，避免路径穿越。
app/services/love_app.py 实现 AI 恋爱大师业务逻辑，构造系统提示词、加载历史、调用模型、保存回复，并在主模型失败时回退到 Ollama。

十、第二阶段 RAG 知识库改造

阶段结论： 第二阶段已完成本地 RAG 知识库 MVP。当前支持读取 data/knowledge 下的 .txt、.md、.json 文件，进行文本切分、embedding、保存本地 JSON 向量索引、检索相关 chunk，并通过 RAG 问答接口返回答案和来源。

1. 新增文件和目录

路径 说明
HuManus/app/rag/loaders.py 加载本地知识文件，并使用 LangChain 文本切分器生成 chunk。
HuManus/app/rag/embeddings.py 封装 OpenAI-compatible、Ollama 和开发用 hash embedding。
HuManus/app/rag/vector_store.py 实现本地 JSON 向量索引和余弦相似度检索。
HuManus/app/services/rag_app.py 实现索引构建、检索和 RAG 增强问答服务。
HuManus/data/knowledge/example.md 新增用于验证的示例知识文件。
HuManus/data/rag/index.json 本地向量索引文件，已加入 .gitignore，避免误提交大文件。

2. 新增配置

RAG_ENABLED=true
RAG_KNOWLEDGE_DIR=./data/knowledge
RAG_INDEX_DIR=./data/rag
RAG_CHUNK_SIZE=800
RAG_CHUNK_OVERLAP=120
RAG_TOP_K=4
RAG_EMBEDDING_PROVIDER=openai_compatible
RAG_EMBEDDING_MODEL=text-embedding-3-small
OLLAMA_EMBEDDING_MODEL=nomic-embed-text
RAG_ALLOW_HASH_EMBEDDING=true
RAG_AUTO_BUILD=true

当前本地 OpenAI-compatible 服务的 chat 接口可用，但 embedding 接口返回 404，因此 RAG 验证时自动回退到了开发用 hash embedding。该回退可验证完整 RAG 流程，但不适合生产语义检索。后续可配置可用的 embedding 服务或 Ollama embedding 模型提升检索质量。

3. 新增接口

接口 说明
POST /api/ai/rag/index 读取本地知识文件并重建 RAG 索引。
GET /api/ai/rag/retrieve?message=... 仅执行检索，返回 top-k chunk、score 和 source，便于调试。
GET /api/ai/rag/chat?message=...&chatId=... 执行知识库增强问答，返回 answer、sources 和 chatId。
