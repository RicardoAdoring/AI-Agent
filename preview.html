yu-ai-agent 改写为 Python 版本任务说明

原仓库：https://github.com/liyupi/yu-ai-agent
目标：将 Java / Spring AI 项目重构为 Python / FastAPI / LangChain / LangGraph 项目
总体结论： 该仓库可以改写为 Python 版本，而且非常适合作为 Python Agent 项目练习。 但不建议逐行翻译 Java 代码，而应该保留原项目的功能思想与接口设计， 使用 Python 生态重新实现后端核心能力。

一、项目现状分析

原项目是一个基于 Java 生态的 AI Agent 项目，核心能力包括： AI 恋爱大师应用、多轮对话、SSE 流式输出、RAG 知识库、工具调用、 MCP 服务接入、ReAct 智能体、PDF 生成、网页搜索与资源下载等。

原后端技术栈主要是： Java 21、Spring Boot 3、Spring AI、 LangChain4j、PGVector、MCP、 SSE、DashScope、Ollama。

原前端是 Vue3 项目，前端本身不一定要重写。只要 Python 后端保持接口路径、 请求参数和响应格式基本一致，前端可以继续复用。

二、改写判断

判断项 结论
是否能改成 Python 可以
是否适合逐行翻译 不适合。应该按功能模块重构。
推荐改写范围 优先重写后端，前端尽量保留。
改写难度 中等偏上，约 7/10。
适合作为什么项目 适合作为 FastAPI + LangChain + LangGraph 的 AI Agent 简历项目。

六、分阶段改写路线

阶段 1：实现最小可运行版本 MVP

  • 搭建 FastAPI 项目。
  • 实现基础聊天接口。
  • 实现 SSE 流式输出接口。
  • 接入 DashScope 或 Ollama 模型。
  • 实现基于 session_id 的多轮对话记忆。
  • 尽量保持原 Vue 前端接口不变。

阶段 2：实现 RAG 知识库

  • 读取 markdown、txt、pdf 等本地知识文件。
  • 进行文本切分。
  • 调用 embedding 模型生成向量。
  • 存入 Chroma、FAISS 或 pgvector。
  • 用户提问时先检索相关内容，再拼接上下文让模型回答。

阶段 3：实现工具调用

  • 实现网页搜索工具。
  • 实现网页抓取工具。
  • 实现文件读写工具。
  • 实现 PDF 生成工具。
  • 实现资源下载工具。
  • 实现 terminate 终止工具。
  • 将这些工具注册到 LangChain / LangGraph Agent 中。

阶段 4：实现 YuManus / ReAct Agent

  • 实现 Agent 状态管理：running、finished、error。
  • 实现最大步数限制。
  • 实现思考、选择工具、执行工具、观察结果、继续规划的循环。
  • 支持模型根据用户目标自主调用工具。
  • 支持调用终止工具后结束任务。

阶段 5：补充 MCP 能力

  • 使用 Python MCP SDK 接入 MCP Server。
  • 将 MCP 工具包装成 Agent 可调用工具。
  • 在 Agent 中统一管理本地工具与 MCP 工具。

七、需要保持兼容的接口

为了尽量复用原前端，Python 后端建议保留类似接口： 

GET /api/ai/love_app/chat/sse?message=xxx&chatId=xxx

GET /api/ai/love_app/chat?message=xxx&chatId=xxx

GET /api/ai/manus/chat?message=xxx

具体请求方式可以根据原前端实际调用代码调整，但优先目标是： 前端尽量不改，后端替换为 Python 实现。

八、核心实现思路

1. LoveApp 应用

Python 版 LoveApp 应负责：构造系统提示词、读取用户历史对话、调用大模型、 支持普通回答、支持流式回答、支持 RAG 增强、支持工具调用。

2. ChatMemory 记忆模块

使用 chat_id 或 session_id 区分不同会话。 初期可以把历史消息保存到本地 JSON 文件，后续再换成 SQLite 或 Redis。

3. RAG 模块

RAG 不需要照搬 Java 的 VectorStore 写法。Python 中可以使用 LangChain 的 DocumentLoader、TextSplitter、Embedding、VectorStore 和 Retriever 重新实现。

4. Tool Calling 模块

每个工具都写成一个普通 Python 函数，再用 LangChain 的 @tool 装饰器包装。Agent 可以根据模型输出决定调用哪个工具。

5. ReAct Agent 模块

可以用 LangGraph 实现，也可以手写循环。核心逻辑是： 

用户目标
-> 模型分析下一步
-> 选择工具
-> 执行工具
-> 获得观察结果
-> 继续规划
-> 达到目标或调用 terminate 工具结束

九、验收标准

  • FastAPI 项目可以正常启动。
  • 基础聊天接口可以调用大模型并返回结果。
  • SSE 接口可以流式输出，而不是一次性输出。
  • 同一个 chat_id 下可以保留多轮上下文。
  • RAG 可以从本地知识库检索内容并辅助回答。
  • Agent 可以根据任务自动调用至少 2 个工具。
  • PDF 工具可以生成文件。
  • 网页抓取工具可以提取网页文本。
  • 前端 Vue 项目可以连接 Python 后端。
